# 매치마인드 시스템 분석 기준 상세 설명

## 📊 시스템 분석 기능 개요

매치마인드는 다양한 분석 기능을 통해 데이터 기반 의사결정을 지원합니다.
모든 분석은 **정량적 지표**와 **AI 기반 정성적 분석**을 결합하여 제공됩니다.

---

## 1️⃣ 인력 추천 분석 (AI 매칭 알고리즘)

### 📌 분석 목적
프로젝트 요구사항과 인력 역량을 다각도로 분석하여 최적의 매칭 제공

### 🔍 분석 기준

#### A. 기술 매칭 점수 (40% 가중치)

**적합도 점수 공식:**
```
Score(P,E) = Σ(Smatch × Wlevel × Wrecency) + (Expdomain × Wdomain)
```

**1. 기본 매칭 (Smatch)**
- 프로젝트 요구 기술과 직원 보유 기술 일치 여부
- 일치: 1.0, 불일치: 0

**2. 숙련도 가중치 (Wlevel)**
- Beginner (초급): 1.0
- Intermediate (중급): 1.5
- Advanced (고급): 1.8
- Expert (전문가): 2.0

**3. 최신성 가중치 (Wrecency)**
- 최근 프로젝트에서 해당 기술 사용 여부 확인
- 시간 감쇠 함수: `e^(-0.3t)` (t = 경과 년수)
- 최근 6개월: 1.0
- 1년 전: 0.74
- 2년 전: 0.55
- 3년 전: 0.41

**4. 도메인 경험 가중치 (Wdomain)**
- 유사 산업군 프로젝트 수행 이력 확인
- 도메인 일치 시: 1.3배 보너스 (30% 가산)
- 예: 금융 프로젝트 → 금융 경험자 우대

**점수 산출 예시:**
```
직원 A: React (Expert, 최근 사용) + 금융 도메인 경험
= 1.0 × 2.0 × 1.0 + (기본점수 × 1.3)
= 높은 매칭 점수
```

#### B. 벡터 유사도 검색 (30% 가중치)

**분석 방법:**
- OpenSearch 벡터 검색 (현재 간소화 버전)
- 프로젝트 요구사항과 직원 프로필의 의미적 유사도 계산
- 단순 키워드 매칭을 넘어선 컨텍스트 이해

#### C. 팀 친밀도 점수 (30% 가중치)

**분석 기준:**
- 기존 팀원들과의 협업 이력
- 과거 프로젝트 공동 참여 경험
- 커뮤니케이션 빈도 및 품질
- 50점 이상: 우수한 팀 시너지 예상
- 0점: 신규 팀원 (친밀도 데이터 없음)

#### D. 가용성 필터

**상태 분류:**
- **Available (즉시 투입 가능)**: 가중치 1.0
- **Busy (다른 프로젝트 참여 중)**: 가중치 0.5
- **Unknown (확인 필요)**: 가중치 0.7

### 📈 종합 점수 계산

**우선순위별 가중치:**

1. **Skill 우선 (기술 중심)**
   - 기술 매칭: 60%
   - 벡터 유사도: 30%
   - 친밀도: 10%

2. **Balanced (균형)**
   - 기술 매칭: 40%
   - 벡터 유사도: 30%
   - 친밀도: 30%

3. **Affinity 우선 (팀워크 중심)**
   - 기술 매칭: 30%
   - 벡터 유사도: 20%
   - 친밀도: 50%

### 🎯 추천 결과 제공 정보

1. **매칭 점수** (0-100점)
2. **일치하는 기술** (녹색 배지)
3. **부족한 기술** (회색 배지)
4. **추가 보유 기술** (파란색 배지)
5. **AI 생성 추천 근거**
   - 강점 분석
   - 약점 분석
   - 추천 이유
   - 주의사항

## 2️⃣ 친밀도 분석 (팀 협업 예측)

### 📌 분석 목적
직원 간 협업 시너지를 예측하여 최적의 팀 구성 지원

### 🔍 분석 기준

#### A. 프로젝트 협업 분석 (35% 가중치)

**측정 항목:**
- 공동 참여 프로젝트 수
- 협업 기간 (개월 수)
- 같은 팀 소속 여부

**점수 계산:**
```
협업 점수 = min(100, 중복 개월 수 × 5)
```

**예시:**
- 6개월 공동 프로젝트: 30점
- 12개월 공동 프로젝트: 60점
- 20개월 이상: 100점

#### B. 메신저 커뮤니케이션 분석 (30% 가중치)

**가중치 공식:**
```
Score(A,B) = Σ(Mt × Wcontext × Wdecay)
```

**1. 메시지 발생 건수 (Mt)**
- 두 직원 간 주고받은 메시지 총 개수

**2. 상황별 가중치 (Wcontext)**
- 업무 시간 (09:00-18:00): 1.0
- 업무 외 시간: 1.5
- 주말: 1.5
- 회사 행사 기간: 2.0
- 연차/휴가 기간: 3.0

**3. 시간 감쇠 (Wdecay)**
- 지수 감쇠 함수: `e^(-0.1t)` (t = 경과 개월 수)
- 최근 1개월: 100%
- 3개월 전: 74%
- 6개월 전: 55%
- 12개월 전: 30%

**응답 시간 보정:**
- 평균 응답 시간이 빠를수록 가점
- 10분 이내 응답: +20점
- 30분 이내 응답: +10점
- 1시간 이내 응답: +5점

#### C. 회사 행사 참여 분석 (20% 가중치)

**측정 항목:**
- 공동 참여 행사 수
- 행사 유형 (팀빌딩, 워크샵, 회식 등)

**점수 계산:**
```
소셜 점수 = min(100, 공동 참여 행사 수 × 20)
```

#### D. 개인적 친밀도 분석 (15% 가중치)

**측정 항목:**
- 월급일 연락 빈도: 15점/회
- 연차일 연락 빈도: 20점/회

**점수 계산:**
```
개인 점수 = min(100, (월급일 연락 × 15) + (연차일 연락 × 20))
```

### 📈 종합 친밀도 점수

**가중 평균 계산:**
```
전체 친밀도 = (협업 × 0.35) + (커뮤니케이션 × 0.30) + (소셜 × 0.20) + (개인 × 0.15)
```

**점수 해석:**
- 80-100점: 매우 높은 친밀도 (최적의 팀 조합)
- 60-79점: 높은 친밀도 (원활한 협업 예상)
- 40-59점: 보통 (일반적인 협업 가능)
- 0-39점: 낮은 친밀도 (팀 빌딩 필요)

## 3️⃣ 인력 평가 분석 (역량 평가)

### 📌 분석 목적
후보자의 역량을 다각도로 평가하여 채용 의사결정 지원

### 🔍 분석 기준 (상대 평가 방식)

#### A. 기술 역량 평가 (30% 가중치)

**평가 요소:**

1. **기술 숙련도 점수**
   - Expert: 10점
   - Advanced: 7점
   - Intermediate: 5점
   - Beginner: 3점

2. **기술 난이도 가중치**
   - AI/ML/Deep Learning: 1.5배
   - Kubernetes/MSA: 1.5배
   - AWS/Azure/GCP: 1.3배
   - Java/Spring/React: 1.2배
   - Python/Node.js: 1.1배

3. **경험 년수 보너스**
   - 기술별 경험 년수 × 0.5점 (최대 5점)

4. **역할 레벨 가중치**
   - Architect: 1.5배
   - Principal: 1.5배
   - Staff: 1.4배
   - Lead: 1.3배
   - Senior: 1.2배

**계산 공식:**
```
기술 점수 = Σ((숙련도 점수 + 경험 보너스) × 난이도 가중치) × 역할 가중치
```

**상대 평가:**
- 모든 직원의 기술 점수 계산
- 최고점자를 100점으로 환산
- 나머지는 비율로 점수 산정

#### B. 프로젝트 경험 평가 (30% 가중치)

**평가 요소:**

1. **프로젝트 규모**
   - 프로젝트 기간 (개월 수)
   - 24개월 이상: 100점
   - 12개월: 50점
   - 6개월: 25점

2. **역할 중요도**
   - Lead/Architect: 100점
   - Principal: 90점
   - Manager: 85점
   - Senior: 80점
   - Developer: 70점
   - Junior: 50점

3. **프로젝트 매칭**
   - 현재 진행 중인 프로젝트와 기술 매칭
   - 매칭 기술 1개당 +2점

4. **프로젝트 개수 보너스**
   - 프로젝트 수 × 5점 (최대 15점)

**상대 평가:**
- 최고점자 기준 100점 환산

#### C. 이력 신뢰도 평가 (20% 가중치)

**평가 요소:**

1. **기술 숙련도 일관성 (40점)**
   - 평균 기술 경험 년수 / 총 경력 년수
   - 일관성 높을수록 고득점

2. **프로젝트 개수 적정성 (30점)**
   - 예상 프로젝트 수 = 경력 년수 × 0.4
   - 실제 프로젝트 수가 예상 범위 내: 30점
   - 너무 적음 (50% 미만): 15점
   - 너무 많음 (300% 초과): 20점

3. **역할-기술 연관성 (30점)**
   - Frontend: React/Vue/Angular 보유 확인
   - Backend: Java/Python/Node.js 보유 확인
   - DevOps: AWS/Kubernetes/Docker 보유 확인
   - 연관성 높음: 30점
   - 연관성 낮음: 15점

**상대 평가:**
- 최고점자 기준 100점 환산

#### D. 문화 적합성 평가 (20% 가중치)

**평가 요소:**

1. **자기소개 키워드 분석**
   - 긍정 키워드: 협업, 소통, 성장, 책임, 열정, 전문가, 리더십, 팀워크
   - 키워드 1개당 +3점

2. **학위 보너스**
   - 박사: +10점
   - 석사: +5점

3. **자격증 보너스**
   - 자격증 1개당 +3점 (최대 15점)

**상대 평가:**
- 최고점자 기준 100점 환산

### 📈 종합 평가 점수

**가중 평균 계산:**
```
종합 점수 = (기술 역량 × 0.3) + (프로젝트 경험 × 0.3) + (이력 신뢰도 × 0.2) + (문화 적합성 × 0.2)
```

### 🎯 AI 생성 분석 결과

**제공 정보:**

1. **강점 분석**
   - 풍부한 실무 경력
   - 다양한 기술 스택
   - 우수한 기술 역량
   - 풍부한 프로젝트 경험

2. **약점 분석 (기술 격차 포함)**
   - 동료 대비 부족한 필수 기술
   - 기술 역량 강화 필요
   - 프로젝트 경험 확대 필요
   - 경력 이력 보완 필요

3. **기술 격차 분석**
   - 같은 직책의 비슷한 경력자들과 비교
   - 50% 이상 동료가 보유한 기술 = 필수 기술
   - 30% 이상 동료가 보유한 기술 = 추천 기술

4. **상세 분석**
   - 기술 스택 분석
   - 프로젝트 유사도 분석
   - 이력 신뢰도 검증
   - 시장 비교 분석

5. **프로젝트 추천**
   - 적합도 점수 기반 상위 3개 프로젝트
   - 추천 이유 및 투입 가능 여부

## 4️⃣ 정량적 분석 (데이터 기반 평가)

### 📌 분석 목적
객관적 지표를 통한 인력 역량 정량화

### 🔍 분석 기준

#### A. 경력 지표 분석

**측정 항목:**

1. **경력 연수 점수**
   - 20년 경력 = 100점
   - 비례 계산: (경력 년수 / 20) × 100

2. **프로젝트 수 점수**
   - 10개 프로젝트 = 100점
   - 비례 계산: (프로젝트 수 / 10) × 100

3. **기술 다양성 점수**
   - 15개 기술 = 100점
   - 비례 계산: (기술 수 / 15) × 100

4. **기술 레벨 분포**
   - Expert/Advanced/Intermediate/Beginner 비율

#### B. 기술 스택 평가

**측정 항목:**

1. **기술 트렌드 점수**
   - TechTrends 테이블에서 각 기술의 트렌드 점수 조회
   - 평균 트렌드 점수 계산

2. **시장 수요 점수**
   - 각 기술의 시장 수요 점수 조회
   - 평균 수요 점수 계산

3. **최신성 점수**
   - 경험 년수가 적을수록 최신 기술
   - 최신성 점수 = max(0, 100 - (경험 년수 × 10))

**종합 기술 스택 점수:**
```
기술 스택 점수 = (평균 트렌드 점수 × 0.5) + (평균 수요 점수 × 0.5)
```

#### C. 프로젝트 경험 점수

**평가 요소:**

1. **프로젝트 규모 점수**
   - 프로젝트 기간 기반 평가
   - 24개월 = 100점

2. **역할 점수**
   - Lead/Architect: 100점
   - Senior: 80점
   - Developer: 70점

3. **성과 점수**
   - 성과 키워드 분석
   - 긍정 키워드: +5점
   - 부정 키워드: -10점

**종합 프로젝트 경험 점수:**
```
프로젝트 점수 = (규모 × 0.3) + (역할 × 0.4) + (성과 × 0.3)
```

### 📈 종합 정량적 점수

**가중 평균 계산:**
```
종합 점수 = (경력 점수 × 0.2) + (프로젝트 수 점수 × 0.15) + (다양성 점수 × 0.15) + (기술 스택 점수 × 0.25) + (프로젝트 경험 점수 × 0.25)
```

## 5️⃣ 도메인 분석 (기술 트렌드)

### 📌 분석 목적
조직의 기술 역량 현황 및 트렌드 파악

### 🔍 분석 기준

#### A. 도메인별 인력 분포

**측정 항목:**
- 각 도메인(분야)별 인력 수
- 평균 경력
- 주요 기술 스택

#### B. 기술 트렌드 분석

**측정 항목:**
- 보유 기술 분포
- 수요가 높은 기술
- 부족한 기술
- 신기술 도입 현황

#### C. 역량 갭 분석

**측정 항목:**
- 프로젝트 수요 vs 인력 공급
- 교육이 필요한 분야
- 채용이 필요한 기술

## 6️⃣ 이력서 분석 (AI 기반 자동 파싱 및 평가)

### 📌 분석 목적
PDF 이력서를 자동으로 분석하여 정보 추출 및 역량 평가

### 🔍 분석 프로세스

#### 1단계: 텍스트 추출 (OCR)

**사용 기술:**
- **AWS Textract** (1차 시도)
  - 고정밀 문서 텍스트 인식
  - 표, 양식 구조 인식
  - 한글/영문 동시 지원

- **PyPDF2** (2차 대체)
  - Textract 실패 시 자동 전환
  - 텍스트 기반 PDF 처리
  - 암호화되지 않은 PDF 지원

**추출 검증:**
- 최소 50자 이상 추출 확인
- 추출 실패 시 명확한 에러 메시지 제공

#### 2단계: AI 기반 정보 추출

**사용 모델:**
- **AWS Bedrock - Claude 3.5 Sonnet**
  - 최신 대규모 언어 모델
  - 한국어 이력서 이해 능력 우수
  - 구조화된 데이터 추출 특화

**추출 정보:**

1. **기본 정보**
   - 이름 (name)
   - 이메일 (email)
   - 직급/직책 (role)
   - 경력 연수 (years_of_experience)
   - 부서/분야 (department)

2. **기술 스택 (skills)**
   ```json
   {
     "name": "기술명",
     "level": "Expert|Advanced|Intermediate|Beginner",
     "years": 경험연수
   }
   ```
   - 기술명 자동 추출
   - 숙련도 자동 판단
   - 경험 년수 계산

3. **정량적 평가 점수 (quantitative_score)**
   - 0-100점 척도
   - 평가 기준:
     - 경력 연수 (30%)
     - 기술 수준 (30%)
     - 프로젝트 경험 (25%)
     - 학력/자격증 (15%)

4. **정성적 분석 (qualitative_analysis)**
   - 강점 분석
   - 약점 분석
   - 특이사항
   - 종합 의견

5. **도메인 전문성 (domain_expertise)**
   ```json
   {
     "금융": 85,
     "클라우드": 90,
     "AI/ML": 75
   }
   ```
   - 주요 도메인 3-5개 추출
   - 도메인별 전문성 점수 (0-100)

#### 3단계: 평가 결과 구성

**종합 점수 계산:**

1. **기술 역량 점수**
   - 기본 점수 + 5점
   - 기술 다양성 및 숙련도 반영

2. **프로젝트 경험 점수**
   - AI 분석 정량 점수 그대로 사용
   - 프로젝트 수 및 규모 반영

3. **이력서 신뢰도 점수**
   - 기본 점수 + 10점
   - 정보 일관성 및 완성도 반영

4. **문화 적합성 점수**
   - 기본 점수 - 5점
   - 자기소개 및 커뮤니케이션 스타일 반영

**종합 평가 점수:**
```
overall_score = (기술 역량 × 0.3) + (프로젝트 경험 × 0.3) + 
                (신뢰도 × 0.2) + (문화 적합성 × 0.2)
```

#### 4단계: 강점/약점 자동 분석

**강점 추출:**
- 보유 기술 스택 수
- 경력 연수
- 전문 분야

**약점 추출:**
- 추가 프로젝트 경험 필요
- 팀 협업 경험 보완 필요
- (AI가 분석한 추가 약점)

**상세 분석:**
- 기술 스택 분석
- 프로젝트 유사도 분석
- 이력서 신뢰도 평가
- 시장 비교 분석

## 7️⃣ 검증 질문 생성 (AI 기반 이력서 검증)

### 📌 분석 목적
이력서의 과장, 허위 기재, 모호한 표현을 찾아내고 면접 검증 질문 생성

### 🔍 분석 기준

#### AI 역할 설정
**20년 차 베테랑 테크니컬 리크루터 + 이력서 검증 감사관**
- 과장 표현 탐지
- 허위 기재 의심 부분 식별
- 모호한 표현 명확화

#### 4가지 검증 관점

**1. STAR 기법 검증**

**분석 대상:**
- 성과(Result)는 명시되어 있으나 과정(Action)이 생략된 부분

**질문 생성 원칙:**
- "어떻게"를 집요하게 묻기
- 구체적인 실행 과정 확인

**예시 질문:**
```
Q: "시스템 성능을 50% 개선했다"고 하셨는데, 
   구체적으로 어떤 병목 지점을 발견했고, 
   어떤 최적화 기법을 적용하셨나요?
```

**2. 기술적 깊이 검증 (Deep Dive)**

**분석 대상:**
- 단순 나열된 기술 스택
- 교과서적 설명만 가능한 수준 의심

**질문 생성 원칙:**
- 교과서적 정의가 아닌 실제 경험 확인
- 트러블 슈팅 경험 (Edge case) 질문
- 기술 선택의 근거 확인

**예시 질문:**
```
Q: Kubernetes를 사용하셨다고 하는데, 
   프로덕션 환경에서 Pod가 OOMKilled 되었을 때 
   어떻게 디버깅하고 해결하셨나요?
```

**3. 기간 대비 성과 검증**

**분석 대상:**
- 경력 연차나 프로젝트 기간에 비해 비현실적인 성과
- 짧은 기간에 과도한 성과 주장

**질문 생성 원칙:**
- 구체적인 구현 난이도 확인
- 시간 배분 및 우선순위 질문
- 팀 규모 및 본인 역할 명확화

**예시 질문:**
```
Q: 3개월 만에 MSA 전환을 완료했다고 하셨는데, 
   몇 명의 팀원과 작업했고, 
   데이터 마이그레이션과 트랜잭션 처리는 어떻게 해결하셨나요?
```

**4. 기여도 분리**

**분석 대상:**
- '팀'의 성과를 '본인'의 성과처럼 포장한 문장
- "우리 팀은 ~했습니다" → "제가 ~했습니다"로 변환된 부분

**질문 생성 원칙:**
- 본인의 정확한 역할 확인
- 팀 내 기여도 비율 질문
- 의사결정 권한 확인

**예시 질문:**
```
Q: "아키텍처 설계를 주도했다"고 하셨는데, 
   구체적으로 어떤 아키텍처 패턴을 제안하셨고, 
   팀 내에서 어떤 반대 의견이 있었으며 어떻게 설득하셨나요?
```

### 📈 질문 생성 결과

**출력 형식:**
```json
{
  "verification_questions": [
    {
      "category": "STAR 기법 검증",
      "question": "구체적인 검증 질문",
      "reason": "이 질문이 필요한 이유 (의심되는 부분)",
      "severity": "high|medium|low"
    }
  ]
}
```

**질문 개수:**
- 총 10개의 검증 질문 생성
- 4가지 카테고리 균형 배분

**심각도 분류:**
- **high**: 허위 기재 의심, 반드시 확인 필요
- **medium**: 과장 표현 의심, 확인 권장
- **low**: 모호한 표현, 명확화 필요

### 🎯 검증 질문 활용 방안

**1. 면접 준비**
- 면접관에게 사전 제공
- 기술 면접 질문 리스트로 활용
- 검증 포인트 사전 파악

**2. 이력서 신뢰도 평가**
- 의심 포인트 개수로 신뢰도 판단
- high severity 질문 많을수록 주의 필요

**3. 후보자 스크리닝**
- 1차 서류 전형 통과 기준
- 검증 질문 답변 요청 가능

**4. 채용 리스크 관리**
- 허위 경력 사전 차단
- 과장 표현 검증
- 실제 역량 확인

## 8️⃣ 도메인 분석 (신규 사업 확장 분석)

### 📌 분석 목적
조직의 현재 역량을 기반으로 진출 가능한 신규 도메인(산업 분야) 발굴 및 확장 전략 수립

### 🔍 분석 프로세스

#### 1단계: 현재 도메인 분류

**데이터 소스:**
- 진행 중(in-progress) 또는 완료(completed) 프로젝트
- 프로젝트의 client_industry 필드
- 프로젝트 이름 및 설명 키워드 분석
- 기술 스택 조합 분석

**도메인 정규화 매핑:**
```python
영어 → 한글 통일
'finance/banking' → '금융'
'healthcare/medical' → '의료'
'e-commerce/retail' → '전자상거래'
'manufacturing/industrial' → '제조'
'logistics/transportation' → '물류'
```

**기술 스택 기반 도메인 추론:**
- Oracle + Spring Boot → 금융
- PostgreSQL + Django + Time Series DB → 제조
- MongoDB + Elasticsearch → 전자상거래
- Cassandra + gRPC → 통신

**결과:**
- 현재 보유 도메인 목록
- 도메인별 프로젝트 수
- 도메인 분류 근거

#### 2단계: 신규 도메인 식별 (TechTrends 기반)

**분석 방법:**

**A. 보유 인력 기술 수집**
```python
employee_skills = {
  'React': 'Expert',
  'Python': 'Advanced',
  'AWS': 'Intermediate',
  ...
}
```
- 모든 직원의 기술 스택 수집
- 기술별 최고 숙련도 추적
- Beginner(1) → Expert(4) 점수화

**B. TechTrends 테이블 분석**

TechTrends 테이블 구조:
```json
{
  "tech_name": "React",
  "category": "Frontend",
  "related_domains": ["전자상거래", "핀테크", "미디어"],
  "growth_rate": 25.5,
  "demand_score": 85,
  "trend_score": 90
}
```

**C. 도메인-기술 매칭**

1. **related_domains 처리** (산업 도메인)
   - 각 기술의 관련 산업 도메인 추출
   - 예: React → 전자상거래, 핀테크, 미디어

2. **category 처리** (기술 도메인)
   - 기술 카테고리를 한글로 정규화
   - 예: Frontend → 프론트엔드 개발
   - AI_ML → AI/ML

3. **기술 매칭률 계산**
   ```
   매칭률 = (보유 기술 수 / 도메인 필요 기술 수) × 100
   ```

**D. 현재 도메인 필터링**
- 이미 보유한 도메인 제외
- 기술 도메인 제외 (프론트엔드, 백엔드, DevOps 등)
- 보유 기술이 있는 도메인만 선택

**E. 기회 점수 계산**
```python
opportunity_score = (
    skill_match_rate × 0.5 +      # 기술 매칭률 (50%)
    growth_rate × 0.2 +            # 시장 성장률 (20%)
    demand_score × 0.2 +           # 수요 점수 (20%)
    trend_score × 0.1              # 트렌드 점수 (10%)
)
```

**결과:**
- 진출 가능한 신규 도메인 목록 (상위 20개)
- 도메인별 기회 점수
- 매칭된 기술 목록
- 시장 지표 (성장률, 수요, 트렌드)

#### 3단계: 도메인 진입 분석

**실현 가능성 점수 계산 공식:**
```
Score(D) = (Trend_growth × W_market) - (Gap_skill × W_risk)
```

**상세 계산:**

**A. 기술 보유율 (Skill Coverage)**
```
기술 보유율 = 보유 기술 수 / 필요 기술 수
```

**B. 인력 가용성 (Employee Availability)**
```
인력 가용성 = min(1.0, 전환 가능 인력 수 / 최소 팀 크기(5명))
```

**C. 기술 갭 페널티 (Skill Gap Penalty)**
```
기술 갭 = 부족한 기술 수 / 필요 기술 수
```

**D. 시장 트렌드 가중치 (W_market)**
```
W_market = (
    (growth_rate / 100) × 0.4 +    # 성장률 (40%)
    (demand_score / 100) × 0.4 +   # 수요 점수 (40%)
    (trend_score / 100) × 0.2       # 트렌드 점수 (20%)
) × 2

범위: 0.5 ~ 2.0
```

**E. 리스크 가중치 (W_risk)**
```
W_risk = 1.0 + (gap_penalty × 0.5)
```

**F. 종합 점수 계산**
```
positive_score = (
    skill_coverage × market_weight × 50 +  # 기술 보유율 (최대 50점)
    employee_availability × 30 +            # 인력 가용성 (최대 30점)
    (trend_score / 100) × 20                # 트렌드 보너스 (최대 20점)
)

negative_score = gap_penalty × risk_weight × 30  # 기술 갭 페널티 (최대 30점)

feasibility = max(0, min(100, positive_score - negative_score))
```

**점수 해석:**
- **70-100점**: 즉시 진입 가능 (높은 실현 가능성)
- **40-69점**: 단기 교육 후 진입 가능 (중간 실현 가능성)
- **0-39점**: 장기 준비 필요 (낮은 실현 가능성)
