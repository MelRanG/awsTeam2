  - 대기 중인 검토: 동적 업데이트

  - 실시간 검색 필터링

  - 기본 정보, 스킬, 경력

  - 성과 지표
- **직원 등록**
  - 모달 기반 등록 폼
  - 실시간 유효성 검사

  - 안전한 삭제 확인 프로세스

### 3. 📋 프로젝트 관리
- **프로젝트 등록**
  - 프로젝트 기본 정보
  - 필요 스킬 및 인력 수
  - 우선순위 설정
- **프로젝트 목록 조회**
  - 상태별 필터링
  - 검색 기능
  - 진행률 표시

### 4. 🤖 AI 기반 인력 추천
- **지능형 매칭 알고리즘**
  - 스킬 매칭 점수
  - 경험 유사도 분석
  - 가용성 고려
- **추천 결과 상세 분석**
  - 매칭 점수 (0-100)
  - 강점/약점 분석
  - 추천 이유 설명
- **프로젝트 배정**
  - 원클릭 배정
  - 배정 이력 관리

### 5. 📄 이력서 평가 시스템

#### 5.1 이력서 업로드 및 분석
- **PDF 이력서 업로드**
  - S3 안전 저장
  - 자동 파싱 및 구조화
- **AI 기반 자동 평가**
  - 기술 스킬 점수 (0-100)
  - 프로젝트 경험 점수
  - 이력서 신뢰도 점수
  - 문화적 적합성 점수
  - 종합 점수 자동 계산

#### 5.2 등록된 직원 검색 및 평가
- **전체 직원 목록**
  - 처음부터 전체 300명 표시
  - 10명씩 자동 스크롤 로딩
  - 실시간 검색 필터링
- **직원별 상세 평가**
  - 기존 직원 재평가 기능
  - AI 기반 역량 분석
  - 성장 가능성 평가

#### 5.3 🔍 이력서 검증 질문 자동 생성 (신규)
- **AI 기반 검증 질문 생성**
  - Bedrock Claude 3.5 Sonnet 활용
  - 이력서 내용 기반 맞춤형 질문
  - 10개 질문 자동 생성 (20-30초)

**검증 질문 카테고리:**
1. **STAR 기법 검증**
   - 성과는 있지만 과정이 생략된 부분
   - 구체적인 상황, 행동, 결과 확인

2. **기술적 깊이 검증**
   - 실제 트러블슈팅 경험
   - 고급 기술 활용 사례

3. **기간 대비 성과 검증**
   - 비현실적인 성과에 대한 구체적 질문
   - 측정 방법 및 지표 확인

4. **기여도 분리**
   - 팀 성과와 개인 기여도 구분
   - 실제 역할 및 책임 범위

**심각도 레벨:**
- 🔴 **High**: 중요한 검증 필요
- 🟡 **Medium**: 보통 수준 확인
- 🟢 **Low**: 참고용 질문

#### 5.4 대기자 명단 관리
- **승인된 지원자 관리**
  - 평가 완료 후 대기자 등록
  - 검증 질문 자동 저장
- **대기자 정보 조회**
  - 평가 결과 상세 보기
  - 검증 질문 리스트 표시
  - 면접관용 질문 활용
- **대기자 삭제**
  - 채용 완료 또는 불합격 처리

### 6. 📈 분석 및 리포트

#### 6.1 도메인 분석

## 🏗️ 시스템 아키텍처

### 프론트엔드
- **React 18 + TypeScript**
- **Tailwind CSS + shadcn/ui**
- **Framer Motion** (애니메이션)
- **React Hook Form** (폼 관리)
- **Sonner** (토스트 알림)

### 백엔드 (AWS Serverless)
- **API Gateway**: RESTful API 엔드포인트
- **Lambda Functions**: 비즈니스 로직 처리
- **DynamoDB**: NoSQL 데이터베이스
- **S3**: 파일 저장소
- **Bedrock**: AI 모델 (Claude 3.5 Sonnet)

### 주요 Lambda 함수
1. `dashboard_metrics` - 대시보드 메트릭
2. `employees_list` - 직원 목록 조회
3. `employee_create` - 직원 등록
4. `employee_delete` - 직원 삭제
5. `employee_evaluation` - 직원 평가
6. `projects_list` - 프로젝트 목록
7. `project_create` - 프로젝트 등록
8. `project_assign` - 프로젝트 배정
9. `resume_upload` - 이력서 업로드
10. `resume_parser` - 이력서 파싱
11. `resume_parse_evaluate` - 이력서 평가
12. `resume_verification_questions` - 검증 질문 생성 ⭐
13. `pending_candidates_list` - 대기자 목록
14. `pending_candidate_delete` - 대기자 삭제
15. `domain_analysis` - 도메인 분석
16. `affinity_calculator` - 친화도 계산
17. `quantitative_analysis` - 정량 분석

## 🚀 주요 워크플로우

### 1. 이력서 평가 및 검증 질문 생성 워크플로우

```
1. 이력서 업로드 (PDF)
   ↓
2. S3 저장 + 파싱
   ↓
3. AI 기반 자동 평가 (20-30초)
   - 기술 스킬 분석
   - 경험 평가
   - 신뢰도 검증
   ↓
4. 평가 결과 표시
   - 종합 점수
   - 세부 분석
   - AI 추천 의견
   ↓
5. 승인/반려 결정
   ↓
6. [승인 시] 검증 질문 자동 생성
   - Bedrock Claude 호출
   - 10개 질문 생성
   - 카테고리별 분류
   ↓
7. PendingCandidates 테이블 저장
   ↓
8. 대기자 명단에서 조회 가능
```

### 2. 프로젝트 인력 추천 워크플로우

```
1. 프로젝트 등록
   ↓
2. 필요 스킬 및 조건 입력
   ↓
3. AI 매칭 알고리즘 실행
   - 스킬 매칭
   - 경험 유사도
   - 가용성 확인
   ↓
4. 추천 결과 생성
   - 매칭 점수
   - 상세 분석
   ↓
5. 프로젝트 배정
```

### 3. 직원 관리 워크플로우

```
1. 직원 등록
   ↓
2. 기본 정보 입력
   ↓
3. DynamoDB 저장
   ↓
4. 직원 목록에 표시
   ↓
5. [필요 시] 직원 평가
   ↓
6. [필요 시] 직원 삭제
```
