# 매치마인드 시스템 시연 가이드

## 🎯 시스템 개요
**매치마인드**는 AI 기반 인력 역량 관리 및 프로젝트 매칭 시스템입니다.
AWS 클라우드 인프라와 AI를 활용하여 최적의 인력-프로젝트 매칭을 제공합니다.

---

## 📋 시연 순서 (추천)

### 1단계: 대시보드 - 전체 현황 파악 (3분)

**목적**: 시스템의 전체적인 현황을 한눈에 보여주기

**시연 포인트**:
- **핵심 지표 카드** (상단 4개)
  - 전체 인력 수
  - 진행 중인 프로젝트 수
  - 투입 대기 인력 수
  - 대기자명단 수

- **인력 활용도** (좌측 상단)
  - 배정률 표시
  - 투입 중/대기 중 인력 현황

- **기술 역량 분석** (중앙 상단)
  - 다재다능 인력 수 (5개 이상 기술 보유)
  - 고유 기술 수
  - 희소 기술 목록

- **액션 필요 항목** (주황색 카드)
  - 대기 인력 수
  - 평가 지연 건수
  - 검증 필요 건수

- **인력/프로젝트 현황 상세**
  - 경력별, 역할별 분포
  - 프로젝트 상태별, 산업별 분포

**설명 멘트**:
> "대시보드에서는 전체 인력과 프로젝트 현황을 실시간으로 확인할 수 있습니다. 
> 현재 X명의 인력이 등록되어 있고, Y개의 프로젝트가 진행 중입니다.
> 특히 주목할 점은 AI가 분석한 기술 역량 분포와 희소 기술 보유자를 확인할 수 있다는 것입니다."

---

### 2단계: 인력 관리 - 직원 등록 및 관리 (5분)

**목적**: 인력 데이터 관리 기능 시연

**시연 포인트**:

#### A. 기존 인력 조회
- 검색 기능 (이름, 기술, 역할)
- 인력 카드에 표시되는 정보
  - 기본 정보 (이름, 직책, 경력)
  - 보유 기술 스택
  - 현재 상태 (투입 가능/투입 중)
  - 평가 점수

#### B. 신규 직원 등록 (모달 시연)
- **기본 정보 입력**
  - 이름, 이메일, 전화번호
  - 직책, 경력 연수
  
- **기술 스택 선택**
  - 다양한 기술 카테고리
  - 숙련도 레벨 설정

- **학력 및 자격증**
  - 학력 정보
  - 보유 자격증

- **프로젝트 경험**
  - 과거 프로젝트 이력
  - 역할 및 성과

#### C. 이력서 업로드 (AI 파싱)
- PDF 이력서 업로드
- AI가 자동으로 정보 추출
- 추출된 정보 확인 및 수정

**설명 멘트**:
> "인력 관리에서는 직원 정보를 등록하고 관리합니다.
> 특히 이력서를 업로드하면 AI가 자동으로 정보를 추출하여 입력 시간을 대폭 단축할 수 있습니다.
> 기술 스택, 경력, 자격증 등 상세한 정보를 체계적으로 관리할 수 있습니다."

---

### 3단계: 인력 평가 - AI 기반 역량 평가 (5분)

**목적**: AI를 활용한 인력 평가 프로세스 시연

**시연 포인트**:

#### A. 대기자명단 확인
- 평가 대기 중인 후보자 목록
- 대기 기간 표시
- 우선순위 정렬

#### B. 평가 프로세스
1. **이력서 평가 시작**
   - 후보자 선택
   - 이력서 내용 확인

2. **AI 자동 평가**
   - 기술 역량 분석
   - 경력 적합성 평가
   - 프로젝트 경험 분석
   - 종합 점수 산출 (100점 만점)

3. **검증 질문 생성** (AI)
   - 이력서 기반 맞춤형 질문 자동 생성
   - 기술 검증 질문
   - 경험 검증 질문
   - 프로젝트 상황 질문

4. **최종 승인/반려**
   - 평가 결과 검토
   - 승인 시 정식 직원으로 등록
   - 반려 시 사유 입력

**설명 멘트**:
> "인력 평가는 AI가 핵심 역할을 합니다.
> 이력서를 분석하여 자동으로 점수를 산출하고, 
> 더 나아가 해당 후보자의 이력서 내용을 기반으로 맞춤형 검증 질문을 생성합니다.
> 이를 통해 면접관이 효과적으로 후보자를 검증할 수 있습니다."

---

### 4단계: 프로젝트 관리 - 프로젝트 등록 및 현황 (4분)

**목적**: 프로젝트 관리 기능 시연

**시연 포인트**:

#### A. 프로젝트 목록 조회
- 진행 중인 프로젝트 카드
- 프로젝트 정보
  - 프로젝트명, 고객사
  - 프로젝트 기간
  - 투입 인력 현황 (X/Y명)
  - 요구 기술 스택

#### B. 투입 인력 상세 정보
- 팀원 목록
  - 이름, 직책, 경력
  - **투입 역할** (프로젝트별 역할)
  - 투입률 (50%, 100% 등)

#### C. 신규 프로젝트 등록
- 프로젝트 기본 정보
- 고객사 및 산업 분야
- 프로젝트 기간
- 필요 인력 수
- 요구 기술 스택
- 우선순위 기술 설정

#### D. AI 인력 추천 버튼
- 인력이 부족한 프로젝트에 표시
- 클릭 시 인력 추천 페이지로 이동

**설명 멘트**:
> "프로젝트 관리에서는 진행 중인 모든 프로젝트를 한눈에 볼 수 있습니다.
> 각 프로젝트의 투입 인력 현황을 실시간으로 확인하고,
> 인력이 부족한 경우 바로 AI 인력 추천을 받을 수 있습니다."

---

### 5단계: 인력 추천 - AI 기반 최적 매칭 (7분) ⭐ 핵심

**목적**: 시스템의 핵심 기능인 AI 매칭 알고리즘 시연

**시연 포인트**:

#### A. 프로젝트 선택
- 드롭다운에서 프로젝트 선택
- 프로젝트 요구사항 자동 표시

#### B. AI 추천 실행
- "AI 추천 받기" 버튼 클릭
- 로딩 중 메시지 표시
- 추천 결과 표시 (3-5초 소요)

#### C. 추천 결과 분석
각 추천 인력 카드에 표시되는 정보:

1. **매칭 점수** (상단 큰 숫자)
   - 0-100점 척도
   - 색상으로 등급 표시 (녹색/노란색/빨간색)

2. **기본 정보**
   - 이름, 직책, 경력
   - 현재 상태

3. **기술 매칭 분석**
   - 일치하는 기술 (녹색 배지)
   - 부족한 기술 (회색 배지)
   - 추가 보유 기술 (파란색 배지)

4. **상세 분석 보기** (모달)
   - **정량적 분석**
     - 기술 매칭률
     - 경력 적합도
     - 가용성 점수
     - 프로젝트 경험 점수
   
   - **정성적 분석** (AI 생성)
     - 강점 분석
     - 약점 분석
     - 추천 이유
     - 주의사항

   - **역량 갭 분석**
     - 부족한 기술 목록
     - 교육 필요 항목
     - 보완 방안

5. **프로젝트 투입** 버튼
   - 투입 역할 선택
   - 투입 기간 설정
   - 투입률 설정 (50%, 100%)
   - 투입 근거 입력

**설명 멘트**:
> "이것이 시스템의 핵심 기능입니다.
> AI가 프로젝트 요구사항과 인력의 역량을 다각도로 분석하여 최적의 매칭을 제공합니다.
> 단순히 점수만 제공하는 것이 아니라, 왜 이 사람을 추천하는지 상세한 분석을 제공합니다.
> 정량적 지표와 함께 AI가 생성한 정성적 분석을 통해 의사결정을 지원합니다."

---

### 6단계: 도메인 분석 - 기술 트렌드 분석 (3분)

**목적**: 조직의 기술 역량 현황 분석

**시연 포인트**:

#### A. 도메인별 인력 분포
- 각 도메인(분야)별 인력 수
- 평균 경력
- 주요 기술 스택

#### B. 기술 트렌드 분석
- 보유 기술 분포
- 수요가 높은 기술
- 부족한 기술
- 신기술 도입 현황

#### C. 역량 갭 분석
- 프로젝트 수요 vs 인력 공급
- 교육이 필요한 분야
- 채용이 필요한 기술

**설명 멘트**:
> "도메인 분석을 통해 조직의 기술 역량 현황을 파악할 수 있습니다.
> 어떤 기술이 부족한지, 어떤 분야에 투자가 필요한지 데이터 기반으로 의사결정할 수 있습니다."

## 🎬 시연 시나리오 예시

### 시나리오: 신규 프로젝트에 최적 인력 배치하기

1. **상황 설정** (대시보드)
   - "현재 15명의 투입 대기 인력이 있습니다"
   - "새로운 프로젝트가 시작됩니다"

2. **프로젝트 등록** (프로젝트 관리)
   - 프로젝트명: "모바일 뱅킹 앱 개발"
   - 필요 인력: 5명
   - 요구 기술: React Native, Node.js, AWS, MongoDB
   - 기간: 6개월

3. **AI 추천 받기** (인력 추천)
   - 프로젝트 선택
   - AI 추천 실행
   - 추천 결과 확인

4. **상세 분석** (추천 상세 모달)
   - 1순위 후보자 선택
   - 매칭 점수: 92점
   - 강점: React Native 5년 경력, 금융 프로젝트 3건 경험
   - 약점: MongoDB 경험 부족 (교육 필요)

5. **프로젝트 투입** (투입 확인)
   - 역할: Frontend Developer
   - 투입률: 100%
   - 기간: 프로젝트 전체 기간

6. **결과 확인** (프로젝트 관리)
   - 프로젝트 카드에서 투입 인력 확인
   - 투입 인력: 1/5명으로 업데이트
   - 팀원 정보에 새로 투입된 인력 표시

## 📊 시연 시 주의사항

### 준비사항
1. 브라우저 캐시 삭제 또는 시크릿 모드 사용
2. 테스트 데이터 확인 (직원, 프로젝트, 대기자)
3. 네트워크 연결 확인
4. 샘플 이력서 PDF 준비

### 시연 팁
1. 각 기능을 천천히 설명하며 진행
2. AI 분석 결과가 나올 때 잠시 대기 (3-5초)
3. 핵심 기능(인력 추천)에 가장 많은 시간 할애
4. 실제 사용 시나리오 기반으로 설명
5. 질문에 대비한 추가 설명 준비

### 예상 질문 대비
- "AI는 어떤 알고리즘을 사용하나요?"
  → 다중 요소 분석 (기술 매칭, 경력, 프로젝트 경험, 가용성 등)
  
- "기존 시스템과의 차이점은?"
  → AI 기반 자동화, 정성적 분석 제공, 실시간 매칭
  
- "확장 가능한가요?"
  → AWS 클라우드 기반으로 무한 확장 가능
  
- "보안은 어떻게 되나요?"
  → AWS IAM, VPC, 암호화 등 다층 보안 적용
